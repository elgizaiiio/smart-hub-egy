

# Files Workspace — Complete Overhaul

## Summary
A comprehensive rebuild of the Files workspace (/files) into a professional, Code-Workspace-style split-pane editor with intelligent agent behavior, deep research, real-time status streaming, proper file exports, and full iOS 26 glassmorphism.

---

## Phase 1: Core Architecture — Split-Pane Layout + Smart Agent

### 1A. Split-Pane Layout (like CodeWorkspace)
Rebuild `FilesPage.tsx` with a desktop split-pane layout:
- **Left panel**: Chat + input (scrollable message area)
- **Right panel**: Live preview (iframe for HTML content, or file-specific preview)
- **Mobile**: Tab switcher between "Chat" and "Preview" tabs (like CodeWorkspace's `activeTab`)
- Header: back button, file title (editable), and action buttons
- All surfaces use `liquid-glass` / `liquid-glass-subtle` classes

### 1B. Smart Agent Behavior
The agent must be intelligent — not just a file generator:
- If user asks a question (no file intent), respond conversationally in chat
- If user requests a file, trigger the deep research + generation flow
- Use intent detection in the system prompt: add a `[MODE: chat | generate]` prefix the AI must output first
- Parse the prefix to decide whether to render a chat reply or trigger file generation

### 1C. Deep Research Flow with Real-Time Status
Replace the current silent `doResearch()` with a streaming research flow:
- Call the `chat` edge function with `searchEnabled: true` and stream status events
- Display real-time steps in `ResearchFlow` component: "Searching for Egyptian history...", "Found 12 sources about pharaohs...", "Gathering data on ancient monuments...", "Writing content..."
- These are **real** status messages from the AI, not hardcoded text
- The system prompt instructs the model to output `[STATUS: ...]` lines that get parsed into ResearchFlow steps
- Research runs for ALL file types (slides, documents, reports, spreadsheets) — not just slides and reports

### 1D. Post-Generation Summary
After file generation, the AI writes a **real** dynamic summary:
- "I created a comprehensive report about Egyptian history covering 5 sections: Ancient Dynasties, The Pyramids, Religious Practices, Daily Life, and Modern Archaeology. The report contains 3,200 words with 4 relevant images. Tap Preview to see it."
- This is generated by the AI based on actual content, not a template

## Phase 2: Smart Questions + Resume Redesign

### 2A. Smart Questions Flow
When user sends a vague request like "Create a report about Egypt":
- AI responds with a Smart Questions card (like ChatPage's SmartQuestionCard)
- Options: "Historical", "Investment", "Cultural", "Tourism" + free text input
- User picks an option → triggers the full research + generation flow
- Implementation: System prompt instructs AI to output `[QUESTION: ...]` blocks when clarification is needed, which get rendered as interactive cards

### 2B. Resume — Full Data Collection
Resume mode gets a complete overhaul:
- When user selects "Resume", show a structured form (not free text):
  - Photo upload, Name, Job title, Email, Phone, Location, LinkedIn
  - Experience entries (company, role, dates, description)
  - Education, Skills, Projects, Languages, Certifications
- Each section is a collapsible card with `liquid-glass-subtle` styling
- Only after all data is collected does the AI generate the resume
- No random/placeholder data — strictly user-provided content

## Phase 3: Preview + Export System

### 3A. New Preview Page
Replace current fullscreen preview:
- **Top bar**: File name + back button (fixed)
- **Middle**: Full preview (iframe)
- **Bottom**: Two floating action buttons — Download + Edit
- Edit button navigates back to chat in same conversation
- All elements use `liquid-glass`

### 3B. Multi-Format Export
When user taps "Download", show a format picker dialog:
- **HTML** — direct download (current)
- **PDF** — use `window.print()` with proper print CSS (current, improved)
- **DOCX** — use `html-docx-js` library (client-side HTML→DOCX conversion)
- **TXT** — strip HTML, export plain text
- For spreadsheets: also offer **CSV** export
- All formats must support Arabic/RTL text properly

### 3C. Remove "Slides Pro" Mode
Remove the "Slides Pro" option from `FILE_SERVICES` array. Keep only:
`slides, document, resume, report, spreadsheet, letter`
